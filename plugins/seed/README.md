# Seed

Turn rough ideas into reviewed specs, decisions, or handoff plans, then stop
before implementation.

Seed is for early product and engineering shaping: clarify intent, compare
approaches, define scope, write a spec or decision artifact, optionally prepare
a compact Writing Plan for complex handoff, review it with the user, and stop.
It is not an implementation runner.

Use Goal Shaper when the task is already clear and needs to become a Codex
`/goal` package.
